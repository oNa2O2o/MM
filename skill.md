## 工作流

### 阶段一：编剧
1. 创作剧情（冲突→升级→悬念三段结构）
2. 写台词（口语化，使用对应地区网络梗）
3. **调用 `/hook-audit` 审核开头钩子**
4. **调用 `/dialogue-check` 检测台词AI味**
5. 与用户确认剧情和台词

### 阶段二：分镜设计

**互动选项原则**：
- 位置由分镜师根据节奏判断（不由用户指定）
- 两个选项都要有爽点，无错误答案
- POV 视角下用第一人称回答

**镜头语言原则**：
- 多机位（过肩/俯拍/仰拍），不越轴（180度规则）
- 中景/中近景为主，特写仅用于情绪高潮
- 第一幕必须建立角色与道具的空间关系（中近景）

**技术预检**：
对照 23 条规则，重点检查：POV 角色不出镜、硬切 ≤ 2 个、无静帧、开头无黑屏

### 阶段三：Seedance提示词
1. **调用 `/seedance-prompt-zh` 生成视频提示词**
2. 提示词不含任何文字、字幕、UI、CTA
3. 台词写入提示词，由角色口述

### 阶段四：交付
1. 导演对照23条规则全量检查
2. **强制输出交付清单**，等用户确认
3. 确认通过后导出 .md 至桌面
4. 命名格式：`YYYY年M月D日_核心题材·情绪关键词·地区受众.md`
5. 询问是否结束任务

### 阶段五：复盘（任务结束后触发）
1. 收集本次修改意见
2. 统一列出规则变动提案，用户一次性回复编号确认
3. 用户确认的编号升级为L3，其余保留L0备注
4. 更新Skill文件版本号
5. 复盘完成后询问是否推送更新到GitHub

## ✅ 23条通用审核规则

### Seedance 技术规范
| # | 规则 |
|---|------|
| T1 | 视频提示词时长 ≤ 15秒，一次性生成 | **为什么**：Seedance 单次生成上限，超时会截断 |
| T2 | POV视角下，主视角角色本身不出现（女性向=女性POV，男性向=男性POV）；提示词中明确写入"禁止POV角色出镜" | **为什么**：POV 是代入视角，主角出镜会破坏沉浸感 |
| T10 | 提示词中明确写入"角色年龄30左右，成熟，禁止幼态角色" | **为什么**：规避审核风险，幼态角色触碰红线 |
| T11 | 提示词中"展示身材"须用具体描述替代模糊表达（如"粗麻囚服贴身，腰线清晰可见"），禁止使用"曼妙身材"等模糊词 | **为什么**：模糊词容易被 AI 理解为色情暗示，具体描述更安全 |
| T4 | 硬切 ≤ 4个，其余用运动镜头 | **为什么**：硬切打断沉浸感，运动镜头保持流畅性，提升完播率 |
| T5 | 无静帧，待机段落为自然微动 | **为什么**：静帧=死画面，会被判定为卡顿，微动保持生命力 |
| T6 | 选项停顿/UI/CTA全部注明为后期制作，不写入提示词 | **为什么**：Seedance 无法生成 UI 元素，写入会导致生成失败 |
| T7 | 第一幕必须建立角色与核心道具同框的空间关系（中近景），禁止角色与道具分离 | **为什么**：建立空间锚点，避免观众困惑"这是哪里" |
| T8 | 分镜须包含多机位（过肩/俯拍/仰拍/正面等），全程不越轴（180度规则） | **为什么**：多机位增加视觉丰富度，越轴会让观众迷失方向感 |
| T9 | 减少特写，以中景/中近景为主；特写仅用于情绪高潮点 | **为什么**：特写过多=压迫感，中景保持舒适观看距离 |

### 开头钩子规范
| # | 规则 |
|---|------|
| H1 | 禁止黑屏开场 | **为什么**：黑屏=秒划走，开头必须有视觉冲击 |
| H2 | 前3秒无声可读，视觉信息足够 | **为什么**：用户刷到时可能静音，纯视觉钩子才能留住人 |
| H3 | 开头已通过 /hook-audit 审核 | **为什么**：开头决定完播率，必须专项审核 |
| H4 | 足够简单，降低用户认知成本 | **为什么**：认知成本高=划走，开头要一眼看懂 |
| H5 | 钩子优化方向禁止使用手机屏幕文字特写（Seedance文字生成不稳定） | **为什么**：Seedance 文字生成不稳定，会出现乱码 |

### 剧情与台词规范
| # | 规则 |
|---|------|
| S1 | 剧情：冲突→升级→悬念，拒绝线性平铺 | **为什么**：线性平铺=无聊，三段式结构制造情绪波动，留住观众 |
| S2 | 台词密度足够，无空镜头堆砌 |
| S3 | 台词通过 /dialogue-check，无AI味 |
| S4 | 使用对应地区网络用语/流行梗 |
| S5 | 整体节奏为连环刺激，非艺术表达 | **为什么**：买量素材目标是转化，不是艺术欣赏，刺激密度决定点击率 |
| S6 | 台词修改只改AI味表达，不得改变原意 |
| S7 | 台词调整必须符合角色身份，不得因去AI味而削弱角色气场 |
